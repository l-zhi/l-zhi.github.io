样式在头部，大dom

样式在头部和底部，大dom

结论：

  • ie和chrome 如果是link放在head中，会加载完head中的link然后构建render tree， 不需要dom全部加载完成。
  • ie link 放在 body中，会从上往下依次加载并构建render tree。所以dom先出现然后变色
  • chrome link 放在 body中则会等待css加载完成后，开始构建render tree， dom这时候不必完全下载完成
  • chrome link 放在 body中时，js也放在body的时候， js 会对render tree的构建产生影响，js以上部分的dom会优先展示出来，然后加载完css之后再重新构建render tree进行最终呈现。
  • 样式都要放在head中，可保持一致性。 render tree的构建，不需要等待dom完成。 只需准备好style rules。
